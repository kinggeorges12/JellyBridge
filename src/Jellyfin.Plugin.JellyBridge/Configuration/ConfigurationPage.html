Library Setup Instructions

Before enabling this plugin, we recommend that you create an empty library. Use the following settings:

  • Content type: 🎬 Mixed Movies and Shows
  • Display name: Anything you'd like! I use "Streaming"
  • Folders: 📁 Use the default folder /data/JellyBridge OR specify the same root folder as found under Library Settings → Library Directory
  • Metadata downloaders (TV Shows): 🔳 Missing Episode Fetcher
  • Metadata savers: ☑️ Nfo
  • ☑️ Save artwork into media folders
  • Subtitle downloaders: 🔳 Open Subtitles (if plugin is active)
🤖 Troubleshooting

The plugin connects to Jellyseerr, but I am seeing error messages when running the sync or other issues.

  1. Keep your plugin up to date! Check the latest release from the Catalog (Jellyfin 10.10) or Plugin (Jellyfin 10.11) page.
  2. Jellyfin uses caching for the plugin configuration pages, so you may need to do a hard refresh of the page to see the latest changes. In Chrome, Open Developer Tools (F12) → Right-click Refresh button → "Empty Cache and Hard Reload".
  3. This plugin is tested for Jellyfin versions from 10.10.7 to 10.11.1. Other versions may not be fully compatible.

The Test Connection button is not working, and I am seeing an error message.

  1. Make sure you have Jellyseerr installed and running. The Jellyseerr URL must be reachable from your Jellyfin server, so check your Docker container and Jellyfin network settings.
  2. Import Jellyfin users from the Jellyseerr interface: Users → Import Jellyfin Users.
  3. Ensure that Jellyseerr is connected to Jellyfin with an admin user. The admin on Jellyseerr's User List should display Jellyfin User under the TYPE column.

The Jellyfin library is not showing images or media, or it is out of sync with the JellyBridge folder contents.

  1. Click the Scan All Libraries button on the Jellyfin dashboard
  2. If content is still not updating, open the Metadata Manager and refresh metadata for the entry matching the name of your Jellyfin library
  3. As a last resort, click Recycle Library Data and recreate the Jellyfin library according to the setup instructions above

Library Settings
Directory path for Jellyseerr library files. Ensure that this path is used only for the JellyBridge library.
After syncing, refreshes all Jellyfin libraries that contain the Library Directory path. Runs a partial refresh "Search for missing metadata" (this should finish quickly), or a full refresh "Replace all metadata" if items are deleted from Jellyseerr (this takes a while). Uses default refresh settings: Replace all images, Keep existing trickplay images.
When enabled, the Jellyseerr library will exclude streaming movies and shows that appear in your Jellyfin libraries. To enable this feature, the scheduled sync task creates an .ignore file in the movie or show directory. The .ignore file contains Jellyfin metadata for reference.
After a request is successfully created in Jellyseerr, the item will be removed from the favorites list for all users.
When enabled, separate libraries will be created for each streaming service, e.g., Netflix, Hulu, FOX.
⚠️ Caution: To display the items in the Jellyfin library without seeing the annoying folder icons, you must add network subdirectories to the Jellyfin Folders rather than the base Library Directory. Additionally, you can use different libraries for each network provider.
Prefix for streaming service library names. The folder structure for the prefix "Network - " would look like:
/data/JellyBridge/Network - FOX/The Matrix (1999)
