ü§ñ Troubleshooting

General debugging tips.

  1. Keep your plugin up to date! Check the latest release from the Catalog (Jellyfin 10.10) or Plugin (Jellyfin 10.11) page.
  2. Jellyfin uses caching for the plugin configuration pages, so you may need to do a hard refresh of the page to see the latest changes. In Chrome, Open Developer Tools (F12) ‚Üí Right-click Refresh button ‚Üí "Empty Cache and Hard Reload".
  3. This plugin is tested for Jellyfin versions from 10.10.7 to 10.11.2. Other versions may not be fully compatible.

The Test Connection with Jellyseerr button is not working, and I am seeing an error message.

  1. Make sure you have Jellyseerr installed and running. The Jellyseerr URL must be reachable from your Jellyfin server, so check your Docker container and Jellyfin network settings.
  2. Import Jellyfin users from the Jellyseerr interface: Users ‚Üí Import Jellyfin Users.
  3. Ensure that Jellyseerr is connected to Jellyfin with an admin user. The admin on Jellyseerr's User List should display Jellyfin User under the TYPE column.

The Jellyfin library is not showing images or media, or it is out of sync with the JellyBridge folder contents.

  1. Click the Scan All Libraries button on the Jellyfin dashboard
  2. If content is still not updating, open the Metadata Manager or Library Manager and refresh metadata for the JellyBridge library
  3. If networks are missing movies and shows, try disabled Manage JellyBridge Library. Run the üì• Import Discover Content from Jellyseerr into JellyBridge Library task and then re-enable the option.
  4. As a last resort, click Recycle Library Data and recreate the Jellyfin library according to the setup instructions above.

The Request JellyBridge Library Favorites in Jellyseerr button is not working, and I am seeing an error message.

  1. Disable CSRF protection in your Jellyseerr instance to allow favorite requests from outside the Jellyseerr interface: Settings ‚Üí Network ‚Üí Uncheck "Enable CSRF Protection".

Import Discover Content
Configure how content is discovered and imported from Jellyseerr. Select which geographic region and streaming networks to pull content from, control how much content is fetched, and set retention policies for discovered items.
Select your region for network services. This determines which watch providers are shown for movies and TV shows.
Shown below as: Jellyseerr sort order #. Watch Provider Name (Provider ID #), and [REGION ID]
Select networks from the right box and move them to the left box. Use the search boxes to filter networks. Active networks will be used for filtering and categorization.
Maximum number of pages to import from the Jellyseerr discover pages for each network. Each page consists of 20 shows for each network. This applies to both movies and TV shows discovery. Enter 0 to retrieve ALL pages (I suggest trying a smaller number first).
Maximum number of days to retain items in the collection before cleanup. JellyBridge items older than this will be removed after running the import task.

Manage Discover Library
Manage how the JellyBridge library interacts with your Jellyfin installation. Configure automatic library refreshing, prevent duplicate content from appearing in multiple libraries, handle favorite removal for requested items, and optionally create separate libraries for each streaming service.
After syncing, refreshes all Jellyfin libraries that contain the JellyBridge Library Directory path. Runs a partial refresh üîÑ Search for missing metadata ‚úÖ Replace all images (this may take more time to retrieve new media data from TMDB and other providers), and an additional full refresh üîÑReplace all metadata if items are deleted from Jellyseerr (this takes less time).
When enabled, the JellyBridge library will exclude streaming movies and shows that appear in your Jellyfin libraries. To enable this feature, the scheduled sync task creates an .ignore file in the movie or show directory. The .ignore file contains Jellyfin metadata for reference.
After a request is successfully created in Jellyseerr, the item will be removed from the favorites list for all users. This setting is to prevent the favorited items from interfering with other plugins like Media Cleaner for Jellyfin. Additionally, this option hides the movie or show in the JellyBridge library by creating an .ignore file in the directory. If you deny the request in Jellyseerr, the item will repopulate on next sync.
When enabled, separate folders will be created for each network service, e.g., Netflix, Hulu, FOX.
Network Folder Setup

Network Folder Setup Instructions

Do this before clicking Sync Discover Content from Jellyseerr to JellyBridge Library or running the Automated Task to Sync Jellyseerr and Jellyfin. This allows the sync task to refresh the libraries after importing discover content.

  1. Select all Network Services that you want to store in the Jellybridge library: Network Services
  2. Click the Generate Network Folders button below.
  3. Create any number of Jellyfin libraries. Each network folder must have a unique Jellyfin library.
  4. For each library, follow the Library Setup Instructions
‚ö†Ô∏è Caution: To display the items in the Jellyfin library without seeing the annoying folder icons, you must add network subdirectories to the Jellyfin Folders rather than the base Library Directory. Alternatively, you can use different libraries for each network provider.
Prefix for network service library names. The folder structure for the prefix "Network - " would look like:
/data/JellyBridge/Network - FOX/The Matrix (1999)
When enabled, duplicate discover content from different networks will be added as long as the content is unique within each library. For example, both HBO and Hulu have "Friends" in their discover content, and both networks are in separate Jellyfin libraries, then each library has a copy of "Friends". The normal behavior is to only import discover content that is unique across all JellyBridge libraries.
‚ö†Ô∏è Caution: You should not include the same network in multiple libraries, as duplicates may occur within libraries.

Sort Discover Content
Control the sort order of items in the JellyBridge library by setting a generated play count of media items for all users. This allows users to see content in a different order each time. Also removes that pesky play count badge next to every media item, so the library no longer shows a that blue circle with a 1 in the upper right corner.
‚ú≥Ô∏è Note: To enable this feature, each user must open the JellyBridge library and change the default sort option Sort by Folder to Sort by Play count (Sort order Ascending).
Enables the automated scheduled task that periodically updates play counts to sort JellyBridge library content.
Defines how often to run the sort task, allowing partial hours, e.g., 1.5. Set to 0 to disable automatic sort randomization. Note: Jellyfin delays scheduled tasks for exactly one hour after first-time install and some version upgrades of the plugin.
Selects the algorithm used to sort JellyBridge library content. None: sets play counts to zero. Random: randomizes play counts. Smart: (semi-)intelligent sorting based on genres in each user's library. Smartish: smart sort with a little randomness.
This feature changes badges to a üîµ‚úì badge. For shows: changes the üîµ1 unplayed count badge. For movies: changes from no badge. Disabling this option changes all media to unplayed badge when sorting.
